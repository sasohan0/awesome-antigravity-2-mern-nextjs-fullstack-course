## Introduction

Welcome to the **Antigravity 2.0 Superuser Bootcamp for MERN Developers**.

This short Eid learning sprint is designed to help you become more than a normal AI tool user. The goal is not just to “generate code with AI.” The goal is to learn how to use Antigravity like a real developer:

- Plan first
- Understand the architecture
- Break large work into small tasks
- Build carefully
- Review every important change
- Verify in the browser
- Test core features
- Document your work
- Ship a portfolio-ready project

Antigravity 2.0 is an agent-first development workspace where AI agents can help with planning, code editing, terminal tasks, browser verification, artifacts, and project workflows.

In this course, we will use Antigravity specifically for **MERN stack and full-stack web development**.

Your goal is simple:

> Use Antigravity as a development partner, not as a shortcut.

---

## Who this course is for

This course is for junior MERN stack developers who already know the basics of web development but want to become stronger, faster, and more professional by using AI-assisted workflows properly.

| Skill | Expected Level |
|---|---|
| HTML, CSS, JavaScript | Basic to intermediate |
| React | Basic component, props, state, and routing knowledge |
| Node.js + Express | Basic API understanding |
| MongoDB | Basic CRUD understanding |

You do **not** need to be an Antigravity expert.

This course starts from beginner-level Antigravity usage and gradually moves toward practical project-building, Figma-to-code workflow, backend planning, testing, deployment, and advanced full-stack thinking.

## Important learning rule

Do not use Antigravity like this:

```txt
Build my full project.
Fix everything.
Make it professional.
```

Use Antigravity like this:

```txt
Create the project plan first.
List the files you will edit.
Implement only the Navbar and Hero section.
Check the browser and report layout issues.
Fix only the broken part.
```

Good developers do not ask AI to “do everything.”

Good developers:

- Give clear tasks
- Review the output
- Make decisions
- Verify results
- Fix mistakes
- Document progress

## Free-plan and quota warning

Antigravity usage can be limited depending on your plan.

This course will teach you how to work efficiently without wasting quota.

You will learn to:

- Use small prompts
- Work one feature at a time
- Save context in markdown files
- Avoid repeating the same explanation
- Ask for plans before code
- Review files before accepting changes
- Verify only after meaningful checkpoints

This is not about bypassing limits.

It is about using your available quota professionally.

---

## Quota-smart workflow

Bad prompt:

```txt
Build the whole project.
```

Better prompt:

```txt
Create only the project plan first.
Do not write code yet.
List the files, commands, implementation order, and risks.
Wait for my approval.
```

Bad prompt:

```txt
Fix all bugs.
```

Better prompt:

```txt
Read only this error and these two files.
Explain the cause.
Suggest the smallest safe fix.
Do not modify unrelated files.
```

Bad prompt:

```txt
Make the UI better.
```

Better prompt:

```txt
Improve only the hero section spacing and mobile layout.
Do not change the navbar, footer, or backend files.
```

Bad prompt:

```txt
Read my full repo.
```

Better prompt:

```txt
Read only PROJECT_CONTEXT.md, API_CONTRACT.md, and src/pages/MenuPage.tsx.
Summarize the issue before editing.
```
