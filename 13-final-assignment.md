# Final Assignment: Design-to-Deploy Full-Stack MERN Product with Antigravity 2.0

**File Name:** `13-final-assignment.md`  
**Assignment Type:** Final Capstone Project  
**Target Learners:** Junior MERN Stack Developers  
**Recommended Duration:** 3–7 focused days  
**Recommended After:** Module 10: Deployment and Production Readiness  
**Main Goal:** Build, test, deploy, and document a real-world full-stack MERN product using Antigravity 2.0 as an agentic development partner while proving that you understand the architecture, codebase, UI, backend, testing, and deployment.

---

## Assignment summary

This is the final project of the **Antigravity 2.0 MERN Developer Edition** course.

You will choose one real-world product idea, use a Figma design or design reference, plan the project properly, build the frontend and backend, connect the full stack, test the app, deploy it, and submit a professional GitHub repository.

This final assignment is not only about creating a website.

It is about proving that you can work like a modern AI-powered MERN developer.

You must show that you can:

- Understand a business problem
- Analyze a design
- Create technical specifications
- Plan architecture
- Use `SKILL.md` files
- Use Antigravity safely
- Build React/TypeScript/Tailwind UI
- Build Express/MongoDB APIs
- Connect frontend and backend
- Test the full project
- Deploy the app
- Write a strong README
- Explain your decisions clearly

---

## Authorization rules

Minimum rules:

```txt
Visitor:
- Can view home/listing/details
- Cannot submit order/booking/request unless logged in
- Cannot access dashboard/admin

Authenticated user:
- Can submit order/booking/request
- Can view own history
- Cannot access admin management

Admin:
- Can manage main resources
- Can view all submissions
- Can update submission status
```

Backend must enforce authorization.

Frontend route guard is not enough.

# Part 20: Common final assignment mistakes

Avoid these mistakes.

## Mistake 1: Building without design

Do not build random UI.

Use a design source or create your own design direction.

---

## Mistake 2: Too much scope

Do not add payment gateway, chat, AI recommendation, maps, real-time tracking, and notifications unless your core project is already complete.

---

## Mistake 3: Weak backend security

Admin routes must be protected on the backend.

Frontend hiding is not enough.

---

## Mistake 4: No testing

A deployed link is not enough.

Submit QA evidence.

---

## Mistake 5: README lies

Do not claim a feature works if it does not.

---

## Mistake 6: No demo credentials

Reviewers should not need to create an admin manually.

Prepare demo credentials safely.

---

## Mistake 7: Committing secrets

Never commit `.env`.

---

## Mistake 8: Not understanding generated code

You must be able to explain the project.

If Antigravity writes code you do not understand, ask it to explain and simplify.
