# Utilize NotebookLM: Learning Guide for Antigravity 2.0 MERN Bootcamp

**File Name:** `01-utilize-notebooklm.md`  
**Module Type:** Learning Workflow / Study System  
**Target Learners:** Junior MERN Stack Developers  
**Estimated Time:** 1.5–2.5 hours  
**Recommended Before:** Module 1: Antigravity Setup  
**Main Goal:** Learn how to use NotebookLM as a structured learning assistant throughout the course.

---

## Module purpose

This module teaches you how to use **NotebookLM** properly during the Antigravity 2.0 MERN Bootcamp.

Antigravity will help you build.

NotebookLM will help you learn.

This is important because AI-assisted coding can become dangerous if you only copy output without understanding:

- What the code does
- Why the architecture is structured that way
- What files are changing
- What errors mean
- What to verify
- What to explain in an interview

NotebookLM will become your course companion for:

- Understanding official docs
- Studying YouTube tutorials
- Creating module-wise notes
- Creating quizzes
- Creating flashcards
- Creating mind maps
- Preparing before using Antigravity
- Reviewing your own project documents
- Turning your learning into interview explanations

The goal is not to use NotebookLM casually.

The goal is to create a repeatable learning workflow.

- Official Antigravity docs

| Mind Map | See the structure of the topic |

| Quizzes | Test yourself |
| Audio Overview | Revise while listening |

NotebookLM is especially useful because it works from the sources you provide.

That means your learning can stay grounded in:

- Official docs
- The course module
- Your project files
- Your selected videos

## Important distinction: NotebookLM vs Antigravity

Do not confuse the two tools.

| Tool | Main job |
|---|---|
| NotebookLM | Learn, summarize, quiz, explain, revise |
| Antigravity | Plan, code, edit files, run terminal commands, verify app |
| GitHub | Save, submit, show proof |
| Figma | Understand design requirements |
| Markdown docs | Preserve project context |

Use NotebookLM before and after development.

Use Antigravity during development.

## When to use NotebookLM

Use NotebookLM at these points:

### 1. Before starting a module

Use it to understand:

- What the module is about
- What you must learn
- What resources are important
- What the practice task requires

### 2. Before asking Antigravity to code

Use it to prepare:

- Project plan
- Required files
- Architecture notes
- Possible mistakes
- Questions to ask Antigravity

### 3. After Antigravity generates output

Use it to review:

- What changed
- Whether the plan still makes sense
- What to verify
- What you should explain in your reflection

### 4. Before submitting work

Use it to create:

- Submission checklist
- Reflection draft
- Quiz review
- Interview explanation

## Important NotebookLM rules

### Rule 1: Add good sources

NotebookLM is only as useful as the sources you add.

Good sources:

- Official docs
- Course modules
- Your project files
- Your own notes
- Selected high-quality tutorials

Weak sources:

- Random screenshots without context
- Low-quality blog posts
- Outdated tutorials
- Unverified social media posts

---

### Rule 2: Ask specific questions

Bad question:

```txt
Explain this.
```

Better question:

```txt
Explain the difference between Antigravity planning, implementation, and browser verification using examples from the sources.
```

Bad question:

```txt
What should I do?
```

Better question:

```txt
Based on this module, create a 90-minute practice plan for a junior MERN developer.
```

Bad question:

```txt
Make notes.
```

Better question:

```txt
Create concise module notes with definitions, examples, mistakes, and action tasks.
```

---

### Rule 3: Use NotebookLM before asking Antigravity

Before asking Antigravity to build, ask NotebookLM:

```txt
Based on these sources, what context should I give Antigravity before starting this task?
```

Then use the answer to create a better Antigravity prompt.

---

### Rule 4: Use NotebookLM after Antigravity output

After Antigravity creates a plan or changes code, ask NotebookLM:

```txt
Review this Antigravity plan/output.

Tell me:
1. What it is trying to do
2. What files are involved
3. What risks exist
4. What I should verify manually
5. What questions I should ask before continuing
```

---

### Rule 5: Do not use NotebookLM as a code executor

NotebookLM is not for running code.

Do not ask NotebookLM to:

- Run your app
- Execute tests
- Install packages
- Verify live deployment
- Fix terminal errors directly

For those tasks, use:

- Antigravity
- Terminal
- Browser
- GitHub Actions
- Deployment logs
