# Contributing

Thank you for improving this course.

This repository is a Markdown-based course for junior MERN/Next.js developers learning Antigravity 2.0, NotebookLM, Figma-to-code, full-stack development, testing, deployment, and portfolio preparation.

---

## What contributions are welcome

You can contribute:

```txt
1. Fix broken links
2. Improve module clarity
3. Add official resource links
4. Add better practice tasks
5. Improve quizzes
6. Improve mentor checklists
7. Add troubleshooting notes
8. Fix typos
9. Add safer Antigravity prompts
10. Update outdated tool instructions
```

---

## What not to contribute

Do not add:

```txt
1. Real secrets or credentials
2. Pirated resources
3. Copyrighted course content copied from others
4. Unsafe terminal commands
5. Unverified tool claims
6. Overly broad "AI will build everything" prompts
7. Links to suspicious packages/tools
8. Adult, gambling, illegal, or unsafe project ideas
```

---

## Contribution style

Keep content:

```txt
Practical
Beginner-friendly
Action-focused
Security-aware
MERN/Next.js relevant
Testable
Honest
```

Avoid:

```txt
Long theory without tasks
Unverified claims
Outdated screenshots as the only instruction
Unclear prompts
Overcomplicated architecture for beginners
```

---

## File naming convention

Use:

```txt
00-introduction.md
01-utilize-notebooklm.md
02-module-01-antigravity-setup.md
...
17-bonus-module-ai-coding-security-safe-agent-workflow.md
18-course-resource-index-and-update-log.md
```

For new files, use lowercase words with hyphens.

---

## Updating resources

When you update a resource link:

1. Prefer official docs first.
2. Add high-quality unofficial resources only if useful.
3. Avoid low-quality clickbait.
4. Add the update to `18-course-resource-index-and-update-log.md`.
5. Mention if the resource is version-sensitive.

---

## Updating module content

A module should generally include:

```txt
1. Purpose
2. Learning outcomes
3. Official/resource links
4. NotebookLM workflow
5. Antigravity prompts
6. Practice tasks
7. Required deliverables
8. Reflection
9. Quiz
10. Mentor checklist
11. Completion standard
```

---

## Pull request checklist

Before submitting a PR:

```md
- [ ] I checked spelling and formatting
- [ ] I did not add secrets
- [ ] I did not add unsafe instructions
- [ ] I used official sources where possible
- [ ] I updated the resource index if needed
- [ ] I kept student workload realistic
- [ ] I checked links
- [ ] I explained the change clearly
```

---

## Suggested PR title format

```txt
docs: improve Module 8 integration troubleshooting
docs: add security checklist to AI agent workflow module
fix: correct broken Vercel documentation link
```

Be respectful, practical, and learner-focused.
